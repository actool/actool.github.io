VTool

A Verification Tool

VTool

Is a model based tool for check conformance between an implementation and specification model.

This tool implement de classical conformance IOCO and a more general conformance based on language that find fault that IOCO don't detect.

The tool was developed in Java and the executable is available for download click here.

Tool Architeture

The rectangles represents the modules of tool, the inclined rectangles are inputs and outputs produced by program, the labels on arrow denotes data exchange between modules.

Graphic library Swing of Java was used in order to create the view module of tool, the accepted inputs are: (i) kind of conformance that is IOCO or based on language; (ii) regex that specify the desirable and undesirable behavior of implementation; (iii) type of model IOLTS or LTS, if IOLTS needs to be informed how distingues the input and output labels; (iv) formal model of implementation and specification in Aldebaran format with .aut extension.

The parses module extracts the IOLTS/LTS model from .aut file verifying if the mode is valid. The automaton construction module get underlying automaton from IOLTS/LTS model and also the automaton that recognize the regex received by view. The last conformance verification module realize operation between automatons and verify the conformance between implementation and specification, the conformance can be IOCO or based on language.

Tool View

Fields:

  • Conformance: conformance verification is based on language or IOCO;
  • Kind of models: the models of implementation and specification are IOLTS ou LTS;
  • Label: if model is IOLTS must inform how distinguish the inputs and output label, it's possible do automatically if .aut file contains label with tag '!' and '?', or inputs and outputs labels can be manually informed;
  • Input/output:this field will appear if is choose inform manually the input and output labels of IOLTS;
  • Implementation:inform the path where the implementation model is in .aut file
  • Model:inform the path where the specification model is in .aut file
  • Language D:can be filled if the conformance based on language is chosen, represents the desired behavior of implementation express in regex;
  • Language F:can be filled if the conformance based on language is chosen, represents the undesired behavior of implementation express in regex;
  • Verify conformance:clicking on the button will verify that the implementation complies with the specification;
  • Result:will inform the verdict whether or not the models conforms
  • Detail:will show the test cases that prove the nonconformity between an implementation and an implementation, in addition to the paths covered by the models when applying each of the test cases
